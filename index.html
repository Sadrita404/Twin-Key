Overview

What I built

This split keyboard is basically built to fix everything annoying about standard typing. It takes the logic of an ortholinear grid, where every key sits in a predictable straight column, and mashes it together with a column stagger that actually matches how long your fingers are.

For long stretches of typing, that combination is exactly the kind of form factor that keeps hands relaxed instead of cramped — which is the whole point of this build.

Motivation

Why this project

Behind choosing this project is basically that I love split keyboards and the way they fold up and stay compact. It's easy to carry, and it's good for writing over long hours — so I wanted to make one for myself, not buy one, and make it fully customisable. That's the reason I chose this project.

Buying a split keyboard off the shelf gets you someone else's decisions baked into the PCB. Designing one from scratch means every trace, footprint, and firmware layer answers to how this person actually types — and gets to be rebuilt the moment that changes.

Features

Everything the board does, from layout to firmware.

01
18×18 split layout

Two independent halves, eighteen keys per side.

02
TRRS halves

Connected with a standard 3.5mm TRRS‑to‑TRRS cable.

03
Dual RP2040 control

RP2040‑Zero on the left, RP2040‑Tiny on the right.

04
Reversible PCB

One board design serves both halves, cutting PCB waste.

05
Wired connection

No wireless link — stable, low‑latency, always on.

06
Ergonomic first

Column stagger and split geometry built around hand shape.

07
QMK firmware

Full QMK support, remappable through VIA.

08
Hot‑swap sockets

Every single key socket is hot‑swappable.

09
Low profile

Slim Kailh Choc switches keep the whole board thin.

10
Custom 3D‑printed case

Case and top plate both designed and printed in‑house.

Setup

Firmware

Precompiled, VIA‑enabled

Flash twin_key_sadrita.uf2 from the Firmware folder onto both RP2040 MCUs to get the board running.

Then load via.json under the Design tab in VIA to remap keys live, no recompiling required.

Full QMK source lives at firmware/QMK/keyboards/Twin‑Key for anyone who wants to go deeper than the config file.

  • Step 1 — Flash twin_key_sadrita.uf2 to the RP2040‑Zero and RP2040‑Tiny.
  • Step 2 — Open usevia.app, go to Design, and import via.json.
  • Step 3 — Remap layers directly in VIA — changes apply instantly.
